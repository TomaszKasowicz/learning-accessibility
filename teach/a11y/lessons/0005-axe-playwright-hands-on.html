Lesson 05 · ~20 min · win: a real, runnable axe scan you control

Hands-on: scanning with @axe-core/playwright

Time to make the engine run. In ~20 minutes you'll stand up a Playwright project, scan a live page against your axe bar (2.2 rule included, correctly enabled), and read real violations — the exact loop your skill automates.

Why this: Lessons 1–4 were the map; this is the vehicle. @axe-core/playwright is the layer-② engine your skill will call. Get it running once by hand and the skill becomes "wrap this in a loop + apply L3 fixes."

1Set up a project

Anywhere you like — a scratch folder is fine:

mkdir a11y-scan && cd a11y-scan
npm init -y
npm i -D @playwright/test @axe-core/playwright
npx playwright install chromium

Versions are released in lockstep with axe-core. Source: @axe-core/playwright README.

2Write the scan

Create a11y.spec.ts. We'll scan Deque's intentionally-broken demo so you actually see violations:

import { test, expect } from '@playwright/test';
import AxeBuilder from '@axe-core/playwright';

test('home page has no axe violations', async ({ page }) => {
  await page.goto('https://dequeuniversity.com/demo/mars/');

  const results = await new AxeBuilder({ page })
    .options({
      // the tag SELECTS rules; rules:{} ENABLES disabled ones (see warning below)
      runOnly: { type: 'tag', values: [
        'wcag2a','wcag2aa','wcag21a','wcag21aa','wcag22aa','best-practice'
      ]},
      rules: { 'target-size': { enabled: true } }
    })
    .analyze();

  // Print a readable summary before asserting
  for (const v of results.violations) {
    console.log(`[${v.impact}] ${v.id} — ${v.nodes.length} node(s)`);
    console.log('  fix: ' + v.helpUrl);
  }

  expect(results.violations).toEqual([]);
});
The trap from Lesson 2, made concrete: withTags(['wcag22aa']) on its own will not run target-size — it's disabled by default and a tag only selects among enabled rules. You must enable it via rules. And in this package .options() overrides withTags()/withRules(), so put both runOnly and rules inside one .options({...}) call (as above) — don't mix the two styles.
Source: axe-core maintainer, issue #3751; playwright README (options overrides).

npx playwright test a11y.spec.ts

That failure is success — you're now reading live violations objects (Lesson 1's anatomy, for real). Each line is impact → rule id → count → helpUrl: enough to triage and fix.

4Three moves you'll use constantly

Scope to a component (include / exclude)

Perfect for Angular component or Storybook tests — scan just what you own, skip third-party noise:

const results = await new AxeBuilder({ page })
  .include('[data-test="date-picker"]')
  .exclude('.third-party-widget')
  .analyze();

Scan state revealed by interaction

axe only sees the current DOM — drive the UI first, then analyze (catches modal/menu bugs):

await page.getByRole('button', { name: 'Open dialog' }).click();
await expect(page.getByRole('dialog')).toBeVisible();
const results = await new AxeBuilder({ page }).include('[role=dialog]').analyze();

Ratchet on a legacy codebase (disableRules)

Too many pre-existing failures to fix at once? Temporarily silence a rule so the gate still protects against new regressions:

const results = await new AxeBuilder({ page })
  .disableRules(['color-contrast']) // TODO: remove once tokens fixed
  .analyze();
Don't forget results.incomplete. Those are items axe couldn't decide — your skill should surface them for the manual layer (Lesson 4), not silently drop them.

5Reusable fixture (so the bar lives in one place)

Encode your whole compliance bar once and reuse it everywhere — this is essentially the skill's automated arm:

// axe-fixture.ts
import { test as base } from '@playwright/test';
import AxeBuilder from '@axe-core/playwright';

export const test = base.extend<{ makeAxe: () => AxeBuilder }>({
  makeAxe: async ({ page }, use) => {
    await use(() => new AxeBuilder({ page }).options({
      runOnly: { type: 'tag', values: [
        'wcag2a','wcag2aa','wcag21a','wcag21aa','wcag22aa','best-practice'] },
      rules: { 'target-size': { enabled: true } }
    }));
  }
});
export { expect } from '@playwright/test';
