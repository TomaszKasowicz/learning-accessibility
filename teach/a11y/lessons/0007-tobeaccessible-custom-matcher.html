Lesson 07 · ~10 min · win: one reusable await expect(...).toBeAccessible()

Wrap your axe scan in a toBeAccessible() matcher

Right now every spec repeats the same three lines: build an AxeBuilder, call .analyze(), then expect(results.violations).toEqual([]). Your R&D doc calls out exactly why that hurts — config duplication and a report that's hard to read. This lesson collapses all of it into a single custom assertion that owns the config once and prints a report you can act on.

Why this matters for the mission. Your fix-wcag skill scans many components. If the compliance bar (the tags + target-size enable) lives inline in every test, it drifts. A toBeAccessible() matcher makes the bar a single source of truth the skill writes once and reuses — and its failure message becomes the structured triage feed your skill reads to apply fixes.

1. The thing we're replacing

This is the level-1 "simple approach" from the doc — and it's what your scan.spec.ts does today:

// repeated in every spec — the two caveats the doc warns about
const results = await new AxeBuilder({ page })
  .withTags(['wcag2a','wcag2aa','wcag21a','wcag21aa','wcag22aa','best-practice'])
  .analyze();
expect(results.violations).toEqual([]);  // ← caveat 2: diff-dumps a giant object, unreadable

Two problems, straight from the R&D doc, p.2 "Caveats":

CaveatWhat it costs you
Duplication of config (tags, disabled rules, excludes)Six tags + the target-size rule re-typed per file; one typo silently lowers the bar.
Report is hard to read/debugtoEqual([]) prints the whole violations array as a deep diff — no impact, no help URL, no selector at a glance.

2. The matcher contract (this is the whole API)

A Playwright custom matcher is just a function you hand to expect.extend. It receives the value passed to expect(...) plus your own args, and it returns an object with two fields:

fieldtypemeaning
passbooleandid the (positive) assertion succeed?
message() => stringlazy string shown when it fails (and on .not success)

That's it. Everything else — name, this.isNot, this.utils — is optional polish. (Playwright docs: expect.extend: "Custom matcher should return a pass flag … and a message callback that's used when the assertion fails.")

The subject can be anything. The official accessibility example passes page. We'll pass { page, testInfo } — same shape your doc's level-3 example uses (expect({ page, testInfo }).toBeAccessible()) — because testInfo lets us attach the full axe JSON to the Playwright HTML report. That's how level 3 fixes the "hard to read" caveat.

3. Build it: AxeBuilder + expect.extend in one file

Create a11y-fixtures.ts. It owns the bar (lifted straight from your scan.spec.ts), runs the scan, and turns violations into a readable message:

// a11y-fixtures.ts
import { test, expect as baseExpect, type Page, type TestInfo } from '@playwright/test';
import AxeBuilder from '@axe-core/playwright';

// the compliance bar — defined ONCE (kills caveat 1)
const BAR = {
  runOnly: { type: 'tag' as const,
    values: ['wcag2a','wcag2aa','wcag21a','wcag21aa','wcag22aa','best-practice'] },
  rules: { 'target-size': { enabled: true } }, // safety net (see lesson 06)
};

type A11yOptions = { include?: string; disableRules?: string[] };

export const expect = baseExpect.extend({
  async toBeAccessible(
    subject: { page: Page; testInfo: TestInfo },
    options?: A11yOptions,
  ) {
    const name = 'toBeAccessible';

    // 1. build the scan from the shared bar + per-call tweaks
    let builder = new AxeBuilder({ page: subject.page }).options(BAR);
    if (options?.include) builder = builder.include(options.include);
    if (options?.disableRules?.length) builder = builder.disableRules(options.disableRules);

    const results = await builder.analyze();
    const v = results.violations;
    let pass = v.length === 0;

    // 2. on failure, attach full JSON to the HTML report (kills caveat 2)
    if (!pass) {
      await subject.testInfo.attach('axe-results.json', {
        body: JSON.stringify(results, null, 2),
        contentType: 'application/json',
      });
    }

    // 3. a short, scannable message — impact, rule, count, help, selector
    const summary = v.map(x =>
      `[${x.impact}] ${x.id} (${x.nodes.length}) → ${x.helpUrl}\n  ${
        x.nodes.map(n => n.target.join(' ')).join('\n  ')}`
    ).join('\n\n');

    if (this.isNot) pass = !pass; // support .not.toBeAccessible()

    return {
      pass,
      name,
      message: () =>
        this.utils.matcherHint(name, undefined, undefined, { isNot: this.isNot }) +
        `\n\n${v.length} violation(s):\n\n${summary}`,
    };
  },
});

export { test };
What you just gained. The bar lives in one BAR const. The assertion is one line. And a failure now prints [serious] target-size (1) → https://dequeuniversity.com/... button.small-button instead of a 200-line array diff — plus the full JSON is downloadable from the report.

Tell TypeScript the matcher exists

expect.extend adds the method at runtime; TS needs a type to match. Augment Playwright's matcher interface (in the same file or a .d.ts):

declare global {
  namespace PlaywrightTest {
    interface Matchers<R> {
      toBeAccessible(options?: { include?: string; disableRules?: string[] }): Promise<R>;
    }
  }
}

Pattern from Playwright's own accessibility-testing guide (shared-config fixture) + the declare global augmentation shown in the Playwright matcher docs.

4. Use it

Import test and expect from your fixture file instead of @playwright/test. The spec shrinks to intent:

// login.a11y.spec.ts
import { test, expect } from './a11y-fixtures';

test('@a11y login page', async ({ page }, testInfo) => {
  await page.goto('/login');
  await expect({ page, testInfo }).toBeAccessible();
});

test('@a11y just the nav, ignoring a known issue', async ({ page }, testInfo) => {
  await page.goto('/');
  await expect({ page, testInfo }).toBeAccessible({
    include: 'nav',
    disableRules: ['color-contrast'], // TODO: JIRA-123, awaiting design
  });
});
The disableRules / include escape hatches map to the doc's p.5 guidance: when a violation can't be fixed yet, skip it per call with a TODO + Jira ref — never by weakening the shared BAR. The bar stays honest; the exception is local and visible.
